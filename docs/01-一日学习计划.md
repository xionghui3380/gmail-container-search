# 一日学习计划（Vue / Java → Next.js 全栈）

> 假设你有 **8 小时** 可用时间。每段包含：**读什么 → 做什么 → 自测**。

---

## 上午：框架概念 + 跑通项目（4 小时）

### 09:00～09:30 | 环境 + 总览

**读：** [README.md](./README.md) → [02-Next.js与React速成](./02-Next.js与React速成(Vue-Java对照).md) 第一、二章  

**做：**
```bash
npm install && npm run db:ensure-parse && npm run db:seed && npm run dev
```
登录系统，依次点开：订单、解析结果、解析日志、仓库汇总、Google Sheet。

**自测：** 能说出本项目是「前后端同仓库、同端口」的全栈结构，而不是 Vue SPA + 独立 Java 后端。

### 09:30～10:30 | Next.js 路由与组件模型

**读：** [02-Next.js与React速成](./02-Next.js与React速成(Vue-Java对照).md) 第三～五章  

**对照打开：**

**做：** 在 `Sidebar.tsx` 里找到五个菜单项，在浏览器里逐一点击，确认 URL 与文件路径对应关系（`/orders` → `app/orders/page.tsx`）。

**自测：**
- App Router 下 `app/orders/page.tsx` 对应 URL 是什么？  
- `LoginForm.tsx` 顶部的 `"use client"` 代表什么？

### 10:30～11:30 | 认证链路（Filter + JWT）

**读：** [04-请求链路详解](./04-请求链路详解(从点击到数据库).md) 第一章  

**对照打开（按顺序）：**

**做：**

3. 用 Postman/curl 无 Cookie 调 `GET /api/v1/auth/me`，应 401  

**自测：** middleware 与 `requireUser` 分别在哪一层、各做什么？为什么 middleware 不能用 Prisma？
